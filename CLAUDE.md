# Claude Safety Rules

## 削除系コマンドの禁止（重要）

以下のルールはこのワークスペース内のすべての会話で絶対に守られる：

- Claude はファイルまたはディレクトリを削除するコマンドを一切生成してはならない。
  例：rm, rm -rf, rm *, rmdir, unlink, cache --delete,
      lftp mirror --delete, rsync --delete, git clean -df, find -delete 等。

- 削除が必要な場合でも、Claude は削除コマンドを提案せず、
  「手動で削除してください」といった説明に留めること。

- 削除の推奨・削除操作の自動判断も禁止。

- ssh / lftp / デプロイ系スクリプトを生成する場合でも、
  削除コマンドの生成は禁止。

これらはすべての会話・コード生成に適用される。

## 開発フロー

### TDD（テスト駆動開発）厳守

順序：plan > red test > coding > green test

- フロントエンド: Jest / Playwright
- バックエンド: RSpec
- フロント動作確認: curl, `wget --mirror`, Playwright のみ使用
- デフォルトアイコン: FontAwesome（絵文字禁止）
- 環境変数: `.env` を参照すること
- 品質チェック: `.claude/DC.md`, `.claude/QC10.md`, `.claude/TM.md`, `.claude/OWASP10.md` を参照
- commitする前に security review を実施すること

### ブランチ戦略

- **mainブランチでの直接作業は禁止**
- `src/*` 以外の変更は main ブランチへの push を許可
- `src/*` の変更は必ず PR を作成すること

### PR規約

- PR本文には非エンジニア向けユーザーテスト手順を丁寧に記載すること
- PR は日本語で記載すること（pr-checker エージェント参照）

## コーディング規約

### 構造

- 制御構文・条件構文以外はクラスまたは関数に書くこと
- セキュリティの観点からグローバル変数を禁止する
- 文字列リテラルは設定ファイルまたはデータベースに分離すること
- ハードコードをチェックするテストを書くこと

### 例外処理

- フォールバック禁止。例外処理をしっかり書くこと
- 例外は握りつぶさず、適切にログ出力・伝播させること

### デバッグトレース

- デバッグトレースできるようにコードを書くこと
- ログレベル（DEBUG / INFO / WARN / ERROR）を明示すること

### UI規約

- ネイティブの `alert()` / `confirm()` / `prompt()` はプロジェクト全体で使用禁止
- デフォルトアイコンは FontAwesome を使用
- 絵文字禁止

## アーキテクチャ

### 技術スタック（基本）

- フロントエンド: Next.js
- バックエンド: Ruby on Rails
- データベース: PostgreSQL
- 必要に応じて: FastAPI（AI・解析・画像加工）、Gin（高速並列・リアルタイム通信）

### 設計方針

- 規模に応じてマイクロサービスアーキテクチャ、MVCアーキテクチャ、API Gateway、メッセージングを意識する
- 安全なライブラリ・フレームワーク・OSS・SaaSを適用し、車輪の再発明を避けてオリジナルコードを少なく保つ

### 多言語対応

- 当初から多言語で開発: 日本語・英語・フランス語・中国語・ロシア語・スペイン語・アラビア語
- 開発者用管理画面は日本語のみ

### 環境判定

- 環境の判定（development / test / production）を必ず実装して分岐できるようにする
- 開発環境では認証済み状態に分岐してテスト可能にすること

## ディレクトリ管理

| ディレクトリ | 用途 |
|---|---|
| `TASKS/` | タスク管理 |
| `DEBUG/` | バグ報告 |
| `CLIENT/` | クライアント要望 |
| `WORK/` | 作業報告 |
| `ENV/DEVELOPMENT.md` | 開発環境情報 |
| `ENV/PRODUCTION.md` | 本番環境情報 |
| `SPEC/` | 仕様書・設計図（ER図・DFD・シーケンス図・クラス図・状態遷移図・ユースケース図） |
| `DELETE/` | ゴミ箱（手動削除対象ファイルを移動する場所） |
| `test/pr***/` | PR別テストスクリプト |

図解: Mermaid を使用すること
